## 问题陈述

### 用户感知

页面在加载过程中出现轻微的内容跳动（布局偏移），滚动时不够流畅，整体使用体验有轻微卡顿感。

### 数据证据

| 指标 | 当前值 | 目标值 | 状态 |
|------|--------|--------|------|
| CLS（累积布局偏移） | **~0.05** | < 0.01 | 需改进 |
| LCP（最大内容绘制） | 194 ms | < 2.5s | 优秀 |
| Render Delay | 155 ms | < 100 ms | 良好 |
| 关键请求链长度 | **668 ms** | < 500 ms | 需优化 |
| 布局偏移次数 | **9 次** | 0 | 需修复 |

### 根因分析（修正后）

1. **CLS 主要来源 — `.site-frame` 的 `padding-inline-start` transition**：
   - `DynamicSidebar.tsx` 在挂载时根据视口宽度设置 `collapsed: false`（[L529-533](file:///f:/codepository/2026311%20132635/Portfolio%20Studio/src/components/DynamicSidebar.tsx#L529-533)）
   - 触发 `syncDockLayout()` → 修改 CSS 变量 `--sidebar-offset`
   - `.site-frame` 的 `padding-inline-start` 从 `0px` 动画过渡到 `~380px`（[global.css#L119](file:///f:/codepository/2026311%20132635/Portfolio%20Studio/src/styles/global.css#L119)）
   - 产生 9 次连续小幅度布局偏移，累积约 0.05

2. **运行时性能 — 强制同步布局（Forced Reflow）**：
   - `DynamicSidebar.tsx` 的 `computeActiveHeading` 中使用 `getBoundingClientRect()`（[L367-368](file:///f:/codepository/2026311%20132635/Portfolio%20Studio/src/components/DynamicSidebar.tsx#L367-368)）
   - 该函数通过 `requestAnimationFrame` 调度，滚动事件监听器使用 `{ passive: true }`
   - **注意**：`getBoundingClientRect()` 是只读 API，不会直接导致 CLS，但会阻塞主线程，影响**滚动帧率**

3. **首屏加载 — Islands 加载策略不当**：
   - `DynamicSidebar` 和 `CompactTimeline` 使用 `client:load` 立即加载
   - 拉长了关键请求链，阻塞了交互就绪时间

4. **动画 GPU 开销**：
   - `.ambient-ribbon` 使用 `filter: blur(18px)` 配合动画
   - `filter` 属性无法被 GPU 加速，在低性能设备上造成压力

---

## 目标

1. 将 **CLS 从 ~0.05 降至 0.01 以下**
2. 将 **关键请求链缩短 30% 以上**
3. 消除滚动过程中的主线程阻塞，保持 60fps
4. 降低低端设备上的动画 GPU 开销

---

## 方案概述

本提案采用四级优化策略，按优先级递进：

| 优先级 | 优化项 | 预期收益 | 关联问题 |
|--------|--------|----------|----------|
| P0 | 移除 `.site-frame` 的 `padding-inline-start` transition | CLS 降至 < 0.01 | CLS 根因 #1 |
| P0 | 将 `DynamicSidebar` 的 `client:load` 改为 `client:idle` | 减少首屏 JS 30%+ | 首屏加载 |
| P1 | 将 `scroll` + `getBoundingClientRect` 替换为 `IntersectionObserver` | 滚动帧率稳定在 60fps | 运行时性能 |
| P1 | 优化 `ambient-ribbon` 动画，移除 `filter` 属性动画 | 减少 GPU 负载 | 动画性能 |
| P2 | 启用 `font-display: swap`（如尚未设置） | 减少字体阻塞渲染 | 字体加载 |
| P2 | 配置 `manualChunks` 分包策略 | 优化缓存命中率 | 构建优化 |

## 风险评估

| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|----------|
| 移除 `padding-inline-start` transition 后 sidebar 展开/收起生硬 | 低 | 低 | 保留 `transform` 动画（sidebar 自身位移），仅移除 `.site-frame` 的 padding transition |
| `IntersectionObserver` 阈值调整不当导致 TOC 高亮不准 | 中 | 中 | 提供多组阈值配置，通过滚动测试验证 |
| `client:idle` 延迟导致侧边栏交互延迟 | 低 | 低 | 使用 `client:idle` 而非 `client:media`，确保空闲时立即加载 |
| 动画调整后视觉效果减弱 | 低 | 低 | 仅移除 `filter` 动画，保留 `transform` 和 `opacity` 动画 |

---

## 验收标准

1. Lighthouse CLS 评分 < 0.01
2. Performance Trace 中页面加载期间无 `Layout Shift` 事件突发
3. 首屏关键请求链 < 500 ms
4. 滚动期间主线程无长任务（> 50ms）
5. 所有现有功能（TOC 高亮、主题切换、语言切换、画廊筛选）正常工作
6. 在 `prefers-reduced-motion: reduce` 下表现正常
