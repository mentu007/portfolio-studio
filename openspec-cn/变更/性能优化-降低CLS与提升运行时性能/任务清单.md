## P0 任务（必须完成）

### 任务 1：消除 CLS — 移除 `.site-frame` 的 padding transition

**根因**：`DynamicSidebar.tsx` 在挂载时展开，触发 `--sidebar-offset` 变化，`.site-frame` 的 `padding-inline-start` transition 产生 9 次连续布局偏移。

- [ ] 在 `SiteLayout.astro` 的 `<script is:inline>` 中，根据视口宽度预设 `--sidebar-offset`
- [ ] 从 `.site-frame` 中移除 `transition: padding-inline-start 320ms ease`
- [ ] 验证 `.site-frame` 的 `padding-inline-start` 在页面首次渲染时即为正确值，无动画过程
- [ ] 验证 sidebar 自身的 `transform` 展开动画仍然正常工作

**关联文件**：
- `src/layouts/SiteLayout.astro`
- `src/styles/global.css`

**验收标准**：
- Performance Trace 中页面加载期间无 `Layout Shift` 事件
- Lighthouse CLS 评分 < 0.01
- sidebar 展开/收起视觉反馈正常

## P1 任务（建议完成）

### 任务 3：重构 DynamicSidebar TOC 追踪（运行时性能）

**根因**：`getBoundingClientRect()` 强制同步布局，阻塞主线程，影响滚动帧率。
**注意**：这是运行时性能问题，不是 CLS 问题。

- [ ] 移除 `scroll` 事件监听器
- [ ] 引入 `IntersectionObserver` 替代 `getBoundingClientRect()`
- [ ] 保留 `pendingScrollTargetRef` 点击导航锁定机制（**不可移除**）
- [ ] 保留 `maxPendingTimeoutRef` 超时释放机制
- [ ] 验证 `prefers-reduced-motion` 下表现正常

**关联文件**：`src/components/DynamicSidebar.tsx`

**验收标准**：
- Performance Trace 中滚动期间无长任务（> 50ms）
- TOC 高亮与滚动位置同步准确
- 点击 TOC 链接后导航正常，高亮保持锁定 1.6 秒
