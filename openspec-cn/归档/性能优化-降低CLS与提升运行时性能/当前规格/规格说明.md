# 规格说明：降低 CLS 与提升运行时性能（修正版 v2）

- **变更名称**：性能优化-降低CLS与提升运行时性能

- **状态**：待审查

## 1. 功能规格

### 1.1 CLS 消除 — `.site-frame` padding 预设

#### 输入

- 视口宽度（`window.innerWidth`）

#### 输出

- CSS 变量 `--sidebar-offset` 的初始值

#### 行为规格

1. **服务器端预设**：在 `SiteLayout.astro` 的 `<script is:inline>` 中，根据视口宽度预设 `--sidebar-offset`
2. **大屏（>= 1180px）**：`--sidebar-offset = 332px`（sidebar 宽度 304px + 间距 28px）
3. **小屏（< 1180px）**：`--sidebar-offset = 0px`
4. **无动画**：`.site-frame` 的 `padding-inline-start` 无 transition，首次渲染即为正确值
5. **客户端调整**：`DynamicSidebar` 挂载后仍可通过 `syncDockLayout()` 调整，但初始值已正确，不会产生布局偏移

---

### 1.2 TOC 高亮追踪（DynamicSidebar）— 运行时性能优化

#### 输入

- `tocItems: TocItem[]` — 目录项数组
- 用户滚动行为（通过 `IntersectionObserver`）
- 用户点击 TOC 链接行为

#### 输出

- `activeId: string` — 当前高亮的目录项 ID

#### 行为规格

1. **初始化**：组件挂载后，为所有 TOC 对应的 heading 元素创建 `IntersectionObserver`
2. **滚动期间**：由 `IntersectionObserver` 回调计算当前 active heading
   - 如果 `pendingScrollTargetRef.current` 存在（点击导航锁定中），忽略 observer 更新
   - 否则按 `boundingClientRect.top` 排序，选择最上方的可见 heading
3. **点击导航**：用户点击 TOC 链接后，设置 `pendingScrollTargetRef = id`，锁定 1.6 秒
4. **锁定释放**：超时后 `pendingScrollTargetRef` 置为 `null`，恢复 observer 驱动的更新
5. **降级**：若 `prefers-reduced-motion` 为 true，滚动行为为 `auto`（无平滑滚动）

#### 状态转换

```
[Idle] ──► [Observer Driving] ──► [Click Locked] ──► [Idle]
              ▲                      │   (timeout)      │
              │                      └──────────────────┘
              │                      (scroll idle)
              └──────────────────────────────────────────┘
```

**注意**：`pendingScrollTargetRef` 是锁定机制的核心状态，**不可移除**。

---

### 1.3 Islands 加载策略

#### 规格

| 组件 | 加载指令 | 触发条件 |
|------|----------|----------|
| DynamicSidebar | `client:idle` | 浏览器空闲时（Astro 内部实现） |
| CompactTimeline | `client:visible` | 元素进入视口 |
| GalleryFilter | `client:visible` | 保持现状 |

**注意**：Astro 的 `client:idle` 具体实现方式未在官方文档中明确说明。

---

### 1.4 动画性能

#### 规格

- `.ambient-ribbon` 仅动画 `transform` 和 `opacity` 属性
- 移除 `filter: blur(18px)` 的实时计算
- 添加 `will-change: transform, opacity` 提示
- `@media (prefers-reduced-motion: reduce)` 下完全禁用动画

## 6. 验收测试用例

### TC-001：无 CLS 页面加载

**前置条件**：清除缓存，打开首页，视口宽度 >= 1180px

**步骤**：
1. 打开 Performance Trace 录制
2. 刷新页面
3. 等待页面完全加载

**预期结果**：
- Trace 中无 `Layout Shift` 事件
- `.site-frame` 的 `padding-inline-start` 首次渲染即为正确值（~380px）

### TC-002：TOC 高亮同步

**前置条件**：进入项目详情页，页面包含多个 heading

**步骤**：
1. 缓慢滚动页面
2. 观察侧边栏 TOC 高亮项

**预期结果**：
- 高亮项与当前视口内的主标题同步
- 无延迟或跳动

### TC-003：点击 TOC 导航锁定

**步骤**：
1. 点击 TOC 中的某一项
2. 页面滚动到对应 heading
3. 观察 TOC 高亮

**预期结果**：
- 页面平滑滚动到目标位置
- 目标项保持高亮至少 1.6 秒（锁定期间）
- 锁定释放后，observer 正常更新高亮

### TC-004：首屏加载性能

**步骤**：
1. 清除缓存，打开首页
2. 运行 Lighthouse 性能审计

**预期结果**：
- CLS < 0.01
- 关键请求链 < 500 ms

### TC-005：减少动画模式

**步骤**：
1. 开启系统"减少动态效果"
2. 刷新页面，滚动浏览

**预期结果**：
- ambient-ribbon 动画停止
- TOC 切换无过渡动画
- 所有功能正常
